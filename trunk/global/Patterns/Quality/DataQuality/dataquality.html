An ontology design pattern for spatial data quality characterization in the Semantic Sensor Web.

An ontology design pattern for spatial data quality characterization in the Semantic Sensor Web. (Ontology) VC Definition

The concepts suggested in this module are not aligned to a foundational ontology because the goal is to provide a simple and readily re-usable set of concepts for semantic annotation. The module aligned to the foundational ontology Dolce Ultra Light (DUL) can be accessed at http://ifgi.uni-muenster.de/~degbelo/Ressources/SSN2012Degbelo. An example of use of the module is provided at http://wsmls.googlecode.com/svn/trunk/global/Patterns/Quality/QualityExample/dataqualityexample.owl.

Data (OWL Class) Definition In use

Data is the output of an observation process involving a sensor, a stimulus, a sensed property and a feature. It is equivalent to the class Observation as defined in the ontology of the W3C's Semantic Sensor Networks Incubator Group (see the definition of 'Observation' at http://purl.oclc.org/NET/ssnx/ssn). Data as used in this ontology refers to spatial data.

The URI of this class is http://www.semanticweb.org/ontologies/dataquality.owl#Data

Data Quality Component (OWL Class) Definition In use

A data quality component is any property of the data that the consumer wants to approximate. Data quality components refer only to aspects of spatial data quality. Examples of data quality components are the accuracy or the resolution of the data.

Definition

The URI of this class is http://www.semanticweb.org/ontologies/dataquality.owl#DataQualityComponent

Data Quality Criterion (OWL Class) Definition In use

A data quality criterion is a criterion defined by the data consumer as meaningful to get information about the quality of the data. It is a measure that helps to approximate the value of the data quality component.

Definition

The URI of this class is http://www.semanticweb.org/ontologies/dataquality.owl#DataQualityCriterion

Data Quality Observation (OWL Class) Definition In use

A data quality observation is an operation by which a data quality value is assigned to a data quality component using a data quality criterion. Data quality as used in this ontology refers only to spatial data quality aspects.
